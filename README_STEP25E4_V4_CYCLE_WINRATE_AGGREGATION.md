# Step25E-4 - V4.0 승률 / 완료주기 집계 재정의 검증판

## 목적

Step25E-3-2에서 V4.0 지정가 원문값 고정과 T/체결 검증은 적용되었지만,
화면의 승률/완료주기 집계가 부분매도 SELL 이벤트 수와 섞여 보이는 문제가 있었다.

V4.0은 한 주기 안에서 1/4 별LOC 매도와 3/4 지정가 매도가 여러 번 발생할 수 있으므로,
SELL 이벤트 개수를 그대로 완료주기수로 사용하면 주기승률이 과대/혼동될 수 있다.

## 변경 내용

- 주기승률은 `CycleId` 기준 완료 주기만 사용한다.
- 완료 주기는 `CycleCompleted=True` 또는 `IsCycleCompleted=True`가 찍힌 CycleId 기준으로 계산한다.
- 한 CycleId 안의 부분매도/지정가매도 손익은 합산하여 하나의 주기 손익으로 판단한다.
- `CycleProfit > 0`이면 수익주기, `CycleProfit <= 0`이면 손실주기로 판정한다.
- SELL 이벤트 기준 승률은 `매도이벤트승률`로 별도 분리한다.
- 대시보드와 Trade Event Log 요약에 완료주기수와 매도 이벤트 수를 분리 표시한다.
- V4.0 검증 섹션에 `Step25E-4 V4.0 승률 / 완료주기 집계 검증` 카드를 추가한다.

## 주의

이 버전은 수익률 개선판이 아니다.
매매 공식, 체결 조건, 지정가 원문값 고정, T 계산 로직은 변경하지 않고,
화면 집계 기준을 주기 단위와 이벤트 단위로 분리하는 검증판이다.
