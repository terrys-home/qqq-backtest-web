# Step25D-7-6 · V2.2 장기주기/OPEN주기 해석판

기준: 사용자가 글로 정리해준 라오어 무한매수법 원문 기준을 유지한다.  
이번 버전은 공식/체결조건/쿼터손절 로직을 새로 바꾸는 단계가 아니라, Step25D-7-5에서 드러난 로그 해석 혼선을 줄이기 위한 UI/표현 정리판이다.

## 변경 목적

- 3번/14번처럼 1000일 이상 이어진 장기 주기가 표시 오류인지 실제 장기 보유 후 완료된 주기인지 더 명확히 보여준다.
- Trade Event Log의 `OPEN`이 “주기 최종상태”가 아니라 “해당 이벤트 직후 상태”임을 구분한다.
- 주기 요약표에서는 `FinalCycleStatus` 관점으로 완료/진행/확인필요를 따로 보여준다.

## 주요 변경

1. `Step25D-7-6 FinalCycleStatus / 장기주기 / OPEN주기 해석판` 추가/개편
   - 최종 COMPLETED
   - 최종 OPEN
   - REVIEW
   - 1000일 이상 장기주기
   - 장기 완료주기
   - 현재 미청산 평가액

2. 장기 주기 상세표 추가
   - 1000일 이상 주기만 별도로 표시
   - 최종상태, 시작일, 종료/기준일, 주기일수, 최종종료사유, 손익, BUY/SELL, 쿼터이벤트 표시

3. 현재 OPEN 주기 상세표 추가
   - 현재 진행 중인 주기만 별도로 표시
   - OpenQty, Open평가액, 최대T, 마지막액션 표시

4. Trade Event Log 문구 정리
   - `주기상태` → `이벤트후상태`
   - `종료사유` → `이벤트후사유`
   - 안내문에 “같은 주기 안에서 OPEN 이벤트가 많아도 최종상태표에서 COMPLETED면 완료 주기”라고 명시
