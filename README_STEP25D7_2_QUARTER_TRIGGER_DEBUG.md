# Step25D-7-2 — V2.2 쿼터손절 발동 디버그/수정판

## 기준

이번 수정은 사용자가 제공한 `라오어 무한매수법 원문` 텍스트를 우선 기준으로 적용했다.
함께 제공된 `라오어 무한매수법 원문.zip`은 원문 사진 묶음으로 확인했지만, 이미지 OCR/추정으로 새 공식을 추가하지 않았다.

## 원문 텍스트에서 확인한 V2.2 쿼터손절 기준

- 40분할 기준 `39 < T <= 40` 또는 40회 매수가 거의 모두 종료되어 더 이상 1회치 매수를 완전히 추가할 수 없는 상태
- 쿼터손절 시작: 누적수량 1/4 MOC 매도
- 이후 남은 자금 + 기존 수익금을 기준으로 10회 분할매수금 계산
- 새 1회 매수금은 기존 1회매수금보다 커지면 안 됨
- TQQQ: -10% LOC 매수/매도 + 10% 지정가
- SOXL: -12% LOC 매수/매도 + 12% 지정가
- 중간 LOC 매도 성공 시 후반전 모드 복귀
- 10회 후 MOC가 반복되면 LOC 매도 상황까지 쿼터손절 반복
- 10회째 MOC 매도가 기준 안쪽이면 후반전 모드 복귀

## Step25D-7-2 변경점

1. 쿼터손절 발동 진단 함수 추가
   - `_quarter_trigger_info()`
   - `QuarterTriggerCandidate`
   - `QuarterTriggerExecuted`
   - `QuarterTriggerReason`
   - `CanAddFullUnitBefore`
   - `RemainingTBefore`

2. 발동 조건 보강
   - 기존: `39 < T <= 40`만 체크
   - 보강: `T >= 39` 근처에서 기존 1회치 매수를 완전히 추가할 현금이 없으면 원문상 “더 이상 1회치 매수를 완전히 추가할 수 없는 상태”로 후보/발동 처리
   - 40분할 외 조건은 자동 적용하지 않고 원문 확인 필요로 로그만 남김

3. 당일 매수 후 소진권 도달 표시
   - 같은 종가 기준으로 LOC 매수와 MOC 매도를 동시에 과최적화하지 않음
   - 당일 매수 후 소진권에 도달하면 `쿼터후보` / `QuarterTriggerAfterActionPendingNextDay`로 표시하고 다음 거래일 첫 액션에서 MOC 시작

4. Trade Event Log 보강
   - `EventSeq` 추가로 실제 로그 생성 순서 보존
   - `QuarterUnitAmount`를 BUY/SELL 이벤트에 표시
   - `ModeTransitionReason`에 쿼터 진입/복귀/반복 사유 표시

5. UI 표시 보강
   - 주기 상세 분석에 쿼터후보/쿼터진입/쿼터MOC/쿼터LOC매수/쿼터LOC매도/복귀/반복 카운터 추가
   - Trade Event Log 요약에 쿼터 카운터 추가
   - Order Plan Log에 쿼터후보/쿼터후보사유 컬럼 추가

## 주의

이번 버전은 쿼터손절 발동과 로그 가시성 보강판이다.
수익률이 좋아지는지보다 먼저 “원문 기준 쿼터손절이 실제로 발동/반복/복귀하는지”를 화면에서 검증하는 것이 목적이다.
